<h1 align="center">Sezione MongoDB</h1>

*MongoDB è un database NO_SQL, che significa Not Only SQL.
Un tempo il database server era costituito da una una singola macchina. Per aumentarne le prestazioni si
ricorreva alla cosiddetta scalabilità verticale cioè l‟aumento di potenza nell‟ambito dello stesso server,
quindi l‟aumento del numero di processori, RAM, dischi, etc. In questa soluzione esiste chiaramente un
limite dettato dalla potenza massima che un singolo computer può raggiungere.
La scalabilità verticale con un unico server che gestisce anche migliaia di utenti contemporanei, è andata
bene fino a quando le connessioni sono rimaste chiuse nell‟ambito dalla rete locale dell‟organizzazione*
